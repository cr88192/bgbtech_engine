Basic Brush

{
faces
...
}

Face:
( x0 y0 z0 ) ( x1 y1 z1 ) ( x2 y2 z2 ) texture [ ux uy uz uo ] [ vx vy vz vo ] ...

The first 3 points define the face plane.
The texture identifies the texture in use.
The numbers in square brackets define the U and V vectors (and the offsets along these vectors).

Alternative Basic Brush

This is another syntax used by the BGBTech engine in basic brushes.

[ nx ny nz nd ] texture [ ux uy uz uo ] [ vx vy vz vo ]

A patch (or Bezier patch) is a grid of points representing a curved surface. The actual surface is derived by interpolating between the points.


Mesh Syntax

{
meshDef1
{
texture ( x y z ) [ ux uy uz ] [ vx vy vz ] mode
(
( ( x y z u v ) ... )
)
...
}
}

Despite the cosmetic similarity to a patch, a mesh-def defines a polygonal mesh, with each row representing a single polygon.
Some information is provided to define the projection of the texture onto the mesh.

Misc Syntax

Cube Definitions:

cubeDef1 ( mx my mz ) ( nx ny nz ) texture ( tx ty tz ) tproj

These provide a more concise syntax to define basic cube-like brushes. These are used in place of a brush definition.
They give a bounding box and a texture origin.
The tproj value gives the texture projection type.


entityDef {
"keyname" "keyvalue"
...
brushes
...
}

This is an alternate entity syntax which may be used for nested entities.
Currently, nested entities are intended for use as "construction entities" for things like duplicators and similar.
They don't imply any structural, mechanical, or behavioral relationship between entities or brushes beyond being able to work with them as a unit for some purpose (such as in a duplicator).
