void bsdeSRand(bsdeUInt seed);

Set seed values for the random number generators.

char *bsdeGetString(int str);

Get a null-terminated string describing some aspect of the implementation.

Name

Description

BSDE_NULL

An empty string.

BSDE_CREATOR

The name of the creator of this implementation.

BSDE_ENGINE

The name of the implementation.
This should fairly uniquely identify the implementation and should not change between implementation versions.

BSDE_ENGINE_VER

The version of the implementation.
If one engine has multiple varieties, (for example, a standalone and networked version, or a software and hw-accel version), these should be encoded in this string, along with any other info that is expected to vary.
The exact structure and meaning of the contents of this string may depend on the implementation.

BSDE_VERSION

The newest version of the specification the implementation conforms to.

BSDE_EXTENSIONS

A space seperated list of known extensions.


The returned value for any unknown string number is to be NULL.

 

World

 


int bsdeNewWorld();
Create a new world context.
At least a single default world is needed for the simulation.
Each world is completely seperate. It is not possible for objects located in different worlds to interact, and handles related to objects in one world are invalid in another (or may refer to a different object).

void bsdeBindWorld(int i);
Set the current world context.
Any objects or joints created will be created within the context of the current world.

void bsdeStepWorld(double dt);
Step the world by a certain amount of time.

Soft-Bodies

 

Soft bodies will occupy the same space as rigid bodies, but will differ greatly in terms of behavior and mechanics.

 

Soft bodies will consist of several arrays:

Vertices, which will hold the current object vertices;

Springs, will hold a number of spring-like relations between vertices;

Faces, will hold a number of faces giving an external structure to the mesh (these will serve as collision geometry).

 

Vertices will each have their own position, velocity, and mass.

Each vertex will have its coordinates stored in world space.

 

When unconstrained, a vertex will move freely in space (subject to gravity, but otherwise non-solid).

 

Vertex Array Properties (IdxGet/IdxSet):

Name

IO

Description

BSDE_VERTEX_ORIGIN

RW,3f

Vertex origin

BSDE_VERTEX_VELOCITY

RW,3f

Vertex velocity

BSDE_VERTEX_MASS

RW,f

Vertex mass

 

Springs will relate 2, 3, or 4 vertices, and apply forces to them.

 

Springs with 2 vertices will generally have linear compression and expansion properties (flexing is not possible with 2 points).

 

Springs with 3 vertices will have both compression and flexing.

 

Springs with 4 vertices will also have volume (they will form a tetrahedron with a defined volume).

 

Springs with 2 vertices will be the same as 3 vertices but have -1 as the last vertex. Vertex 0 will be special and refer to the object's origin.

RW,f

Attenuation factors used for maintaining the volume of tetrahedral springs.

 

 

 

These will define the ideal shape of the spring. Note that area and volume will be aliased, and will be defined by the lengths in question. Many of these values will be computed when creating the spring.

 

 

The attenuation factors will effect how strongly the the spring will attempt to retain its ideal shape. Likewise, the area and volume attenuations will be aliased.
