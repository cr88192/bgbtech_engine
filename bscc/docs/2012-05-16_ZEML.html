ZEML: Z-Expression Markup Language

ZEML is a sort of hybid of XML and S-Expressions.
It operates within a similar model as XML, but has a tag structure vaguely more like that of S-Expressions.

Like in XML, pointy brackets are used.


Numbers

Numbers and Strings will be partially distinct in ZEML.
It is not required that implementations be able to distinguish them, however the representation of a value as a number may serve as a hint to use more efficient storage. Similarly, a number need not preserve its exact representation, but need only preserved by value. If a number can't be represented within an available numeric type, it should be instead handled as a string.


Arrays

Arrays will be another extension. An array will exist either as a standalone type, or as a string containing its items (including braces). An implementation is not required to be able to distinguish an array from a string. Like with numbers, arrays will exist as an optimization hint.

Syntax



Node
    Tag | QuotedText | BlockText | Value

Tag
    <nsname Attribute* Node*>

Attribute
    nsname '=' value

Value
    name | number | QuotedText | Array

Array
    [ Value+ ]

QuotedText
    "..."

Quoted Text will consist of text in quotes.
This will support a number of C-like escapes:
    \n    Newline
    \t    Tab
    \r              Carriage Return
    ...
    \xHH        2-digit character.
    \uHHHH    4-digit character
    \UHHHHHHHH    8-digit character.

In addition, text may contain newlines, which will be embedded into the parsed text.
If there is a '\' followed by a newline, then the newline will be ignored.

BlockText
    <[[ ... ]]>


name = NameInitChar NameChar*
NameChar = NameInitChar | Digit
NameInitChar = a-z | A-Z | _

nsname = ( name : )* name

digits = digit ( digit | _ )*
digit = 0-9
hexdigit = digit | A-F | a-f
hexdigits = hexdigit ( hexdigit | _ )*

number = 0x hexdigits | [ . digits ] [ (E|e) (+|-) digits ]
