BGBScript 2B

Goals:
  • Possible: Reboot of BGBScript and BGBScript2 efforts.
  • Will be statically typed and performance oriented.
  • Will change the canonical declaration syntax.
  • Will use a class/instance object system.


Scope Levels:

  • Global
  • Package
  • Class
  • Function/Method
  • (Possible) Lexical


Objects

  • Single Inheritance
  • Interfaces
  • Reference and Value Classes
  • Reference and Value Structs
  • COM-like Object System
    • Objects may be exported to C as structs
    • Will have C-callable VTables
      • A trick will be used to make this work for plain C interpreters.
      • VM internal VTables may be separate.

VM

  • Statically-Typed Register IR

Modifiers:
abstract Method is incomplete.
Class can't be instantiated directly, but must be overloaded.
const Value is immutable.
dynamic Class: May allow addition of new members.
extern Reserved
final Class: May not be overloaded.
Variable is immutable.
native import: Specifies a native library.
package: Specifies a package to be exported.
class or struct: Should be exportable to C.
private Only accessible from the same class.
protected Only accessible from the same class or child classes.
Default access type in class.
public Accessible globally.
Default access type for struct.
register Reserved
static Fields: Will share the same value between all instances.
Method: Method does not depend on or use the object instance.
Locals: The local will be shared between all calls to this function or method.
synchronized Lock accesses with a mutex.
virtual Method may be overloaded.
